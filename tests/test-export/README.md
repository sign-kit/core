# test-export

A minimal Vue 3 app used to validate the `@signkit/core` package as a real consumer would use it — testing component rendering, IntelliSense, and style imports against the built tarball.

## How it works

The project installs `@signkit/core` from a local `.tgz` tarball (`../tmp-pack/sign-kit-core-0.1.0.tgz`) rather than from npm. This lets you validate the exact package contents that would be published.

## Workflow: build → pack → test

Run these commands from the **monorepo root** (`c:\Developer\signkit-core`):

```sh
# 1. Build the library (types, ESM/CJS bundle, WC bundle, styles)
npm --workspace=@signkit/core run build

# 2. Pack it into a tarball in tmp-pack/
npm pack --workspace=@signkit/core --pack-destination tmp-pack

# 3. Install the tarball into this test app
cd test-export
npm install ../tmp-pack/sign-kit-core-0.1.0.tgz
```

Then start the dev server (from `test-export/`):

```sh
npm run dev
```

## IntelliSense / types

Types are generated by `vue-tsc` during `build:types` and included in the tarball. After installing the tarball, VS Code (with the [Vue - Official](https://marketplace.visualstudio.com/items?itemName=Vue.volar) extension) will provide full IntelliSense for `FormBuilder`, `Signer`, and all exported types.

If types appear missing, verify the installed package contains `.d.ts` files:

```sh
# Verify installed package
ls node_modules/@signkit/core/dist/components/FormBuilder/
# Should include: FormBuilder.vue.d.ts
```
